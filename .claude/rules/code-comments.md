# コード内コメント規約

## ドキュメント参照のリンク形式

コード・設定ファイル内でドキュメントを参照する場合、Markdown リンク形式で記述する。

```rust
//! 詳細: [BFF パターン](../../../docs/05_技術ノート/BFFパターン.md)
```

```yaml
# 技術詳細: [GitHub Actions](../../docs/05_技術ノート/GitHubActions.md)
```

```javascript
/**
 * 技術詳細: [Vite](../../docs/05_技術ノート/Vite.md)
 */
```

パスは参照元ファイルからの相対パスで記述する。

## コメントの原則

- コード内のコメントは「何をしているか」を簡潔に
- 「なぜ」「どう使うか」の詳細解説は技術ノートに書き、コードからリンク
- 設定ファイルの詳細説明は ADR や技術ノートに委譲し、設定ファイル自体は簡潔に保つ
