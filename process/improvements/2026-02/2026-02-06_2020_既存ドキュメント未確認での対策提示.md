# 既存ドキュメント未確認での対策提示

## 事象

Docker ビルドで `elm` コンパイラが見つからないエラーが発生した際、手順書（開発環境構築.md）を確認せずに `package.json` に `elm` を追加しようとした。

手順書には「Elm ツール群はグローバルにインストールする（ADR-002 参照）」と明記されており、正しい対策は Dockerfile で `npm install -g elm` することだった。

## 原因分析

1. エラー発生 → 即座に修正案を出す「目先の対策に飛びつく傾向」の再発
2. CLAUDE.md に「既存ドキュメントの確認を必須とする」と明記されているが、問題解決の焦りで省略した
3. `elm-review` と `elm-test` が `devDependencies` にあることから、`elm` も同様に追加すべきと推測した（実際は設計判断で分けられていた）

## 対策

1. **問題解決フレームワークの徹底**: エラーが発生しても即座に修正案を出さず、まず Want → To-Be → As-Is → ギャップ分析の流れに従う（既存ルール、追加対策不要）
2. **既存ドキュメントの確認**: 対策を提示する前に、関連する手順書・ADR を確認する（既存ルール、追加対策不要）

## 次のアクション
