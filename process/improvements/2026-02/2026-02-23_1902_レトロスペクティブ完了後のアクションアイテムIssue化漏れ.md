# レトロスペクティブ完了後のアクションアイテム Issue 化漏れ

## 事象

`/retro` 実行後、レポートを保存して PR を作成・マージしたが、Issue #791 のチェックリストに含まれる「アクションアイテムから必要な Issue を作成」ステップを実行しなかった。レポート保存後に「Issue 化が必要であればお知らせください」とユーザーに判断を委ね、その後 PR 作成・レビュー・マージのフローに注力して Issue 化ステップを忘れた。

## 原因分析

### 直接原因

レトロスペクティブの実施 Issue（#791）のチェックリストを確認せずにフローを進めた。レポート保存後、PR 作成に移行する際に残タスクの確認を省略した。

### 根本原因

`/retro` スキルの定義には「レポートの保存」と「アクションアイテムを Issue 化する際のラベル付与」は記載されているが、「アクションアイテムの Issue 化」自体がスキル定義のステップとして組み込まれていない。スキル実行後のタスクが実施 Issue のチェックリストにのみ依存しており、スキル定義側で完結しない構造になっている。

## 対策

問題の性質: プロセス的（スキル定義の完結性不足）

### 対策形式の検証

| 対策案 | 形式 | 有効性 |
|--------|------|--------|
| ~~「レポート保存後にチェックリストを確認する」~~ | 行動規範 | 低 |
| `/retro` スキル定義にアクションアイテム Issue 化ステップを追加し、スキル内で完結させる | フロー組み込み | 高 |

### 採用する対策

`/retro` スキル定義の「レポートの保存」セクションの後に、アクションアイテムの Issue 化ステップを追加する。スキル定義内でフローが完結するため、実施 Issue のチェックリストへの依存が不要になる。
