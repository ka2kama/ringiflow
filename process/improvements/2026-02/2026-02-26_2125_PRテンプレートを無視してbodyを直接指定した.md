# PR テンプレートを無視して body を直接指定した

## 事象

PR 作成時に `gh pr create --body` でテンプレート（`.github/pull_request_template.md`）の構造を無視した本文を直接指定した。テンプレートで定義された品質確認カテゴリ（ドキュメント変更用の差し替え版）が使用されず、独自の構造で品質確認を記載しようとした。

ユーザーの指摘を受け、テンプレート構造に沿った本文で再作成した。

## 原因分析

CLAUDE.md に「AI エージェントは `--body` で本文を直接指定し」と記載されている。この指示を「テンプレートを参照せずに自由に本文を書いてよい」と解釈した。

実際には「`--body` フラグを使用する（`--fill` やエディタ起動ではなく）」という技術的指示であり、テンプレートの構造に従うことは前提条件だった。

根本原因: CLAUDE.md の記述が `--body` の使用方法（How）のみを指定しており、テンプレート準拠（What）を明示していない。

## 対策

CLAUDE.md の PR 本文ルールに、テンプレート準拠を明示する。

現状:
```
AI エージェントは `--body` で本文を直接指定し、末尾に署名
```

対策案:
```
AI エージェントは PR テンプレート（`.github/pull_request_template.md`）の構造に沿った本文を `--body` で直接指定し、末尾に署名
```
