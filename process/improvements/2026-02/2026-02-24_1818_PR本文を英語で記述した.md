# PR 本文を英語で記述した

## 分類

- カテゴリ: ドキュメント規約違反
- 失敗タイプ: ルール遵守漏れ（繰り返し発生）
- 問題の性質: プロセス的

## 事象

PR #895 の本文（Summary、Test plan セクション）を英語で記述した。CLAUDE.md の段階的英語化ルールでは PR 本文は日本語と定められている。同様の問題は過去にも複数回発生している。

## 原因分析

### Want

PR 本文が日本語で記述されること（段階的英語化ルール）。

### To-Be

PR テンプレートと AI の出力が自然に日本語になる構造。

### As-Is

PR テンプレート（`.github/pull_request_template.md`）のセクション名が英語（`## Summary`, `## Test plan`）で記述されている。AI はテンプレートのセクション名に引きずられて本文も英語で記述してしまう。

### ギャップ分析

テンプレートのセクション名が英語であるため、AI がセクション名の言語コンテキストに引きずられて本文も英語で書く。CLAUDE.md のルール（PR 本文は日本語）よりも、直近のコンテキスト（英語のセクション名）の方が AI の出力言語に影響を与えている。

### 根本原因

PR テンプレートのセクション名が英語であり、AI がテンプレートの言語コンテキストに従って本文を英語で生成するパターンが構造的に存在する。行動規範（「日本語で書くこと」）だけでは繰り返し発生する。

## 対策

テンプレート自体を日本語にすることで、AI が自然に日本語で本文を記述するようにする（フロー組み込み）。

| 対策 | 形式 | 内容 |
|------|------|------|
| PR テンプレートのセクション名を日本語化 | フロー組み込み | `## Summary` → `## 概要`、`## Test plan` → `## テスト計画` に変更 |

行動規範からの転換: 「PR 本文は日本語で書くこと」（行動規範）→ テンプレート自体が日本語であることで自然に日本語出力を誘導（フロー組み込み）。
