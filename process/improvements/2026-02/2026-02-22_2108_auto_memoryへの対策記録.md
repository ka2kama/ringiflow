# auto memory への対策記録

## 事象

#769 の実装後、「openapi.yaml を手動編集した」という指摘に対し、MEMORY.md（auto memory: `~/.claude/projects/.../memory/`）に対策を書こうとした。MEMORY.md を使わないという過去の指摘があったが、セッションをまたいで情報が失われていた。

## 原因分析

### 表層的な原因

「MEMORY.md を使わない」という指摘が改善記録 `2026-02-05_2104_目先の対策に飛びつく傾向.md` に事象として記録されているが、ルールとして明文化されていない。

### 根本原因

「auto memory は永続化手段として使用しない」がルールファイルや CLAUDE.md に記載されていないため、セッションをまたぐと同じ行動を繰り返す。改善記録の事象から教訓を抽出してルール化するプロセスが不完全。

## 対策

- 問題の性質: プロセス的
- CLAUDE.md またはルールファイルに「auto memory は永続化手段として使用しない」を明記する
- 対策形式の検証: 成果物要件（ルールの存在）で転換済み
- Issue #772
