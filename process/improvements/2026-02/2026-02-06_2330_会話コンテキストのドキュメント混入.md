# 会話コンテキストのドキュメント混入

## 発生日時

2026-02-06

## 概要

会話中の主観的評価がそのまま GitHub Issue の本文に混入した。

## 事象

独自フレームワークと既知手法の比較について議論した後、idea Issue を作成した際、会話中の評価（「筋は良い」等）がそのまま Issue 本文に含まれた。

ユーザーから主観的評価を Issue に含めないよう指摘を受けて修正。

## 根本原因

1. 会話モード（対話的・主観的評価を含む）とドキュメント作成モード（事実ベース・読者に依存しない記述）の切り替えが明示されていない
2. Issue 作成時に、直前の会話内容をそのまま流用する傾向がある。会話の文脈を共有しない読者の視点が欠落している

## 対策

ドキュメント（Issue、PR、設計書等）を生成する際、以下を自己チェックする:

- この文章は会話の文脈を知らない読者が読んでも意味が通るか？
- 主観的な評価（「良い」「筋が通っている」「素晴らしい」等）が混入していないか？
- 事実と評価が分離されているか？

## 学び

- 会話での応答とドキュメントの記述は、読者と目的が異なる。会話は相手との対話、ドキュメントは不特定の読者への情報伝達
- AI は会話の流れに乗りやすく、直前のコンテキストをそのまま引きずる傾向がある。モード切り替えを意識的に行う必要がある
