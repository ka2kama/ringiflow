# 検証セクションの効果を対策実施内容で記載

## 事象

改善記録 `2026-02-14_2137_改善記録のカテゴリと失敗タイプの混同.md` の検証セクションで、「効果」フィールドに対策の実施内容（「README に照合手順を追加。定義済み値の一覧と禁止事項を明記し…」）を記載した。

README の定義では「効果」は「同種の問題が再発していないか」を確認するフィールドであり、対策実行直後に記入できるものではない。

## 原因分析

### 表層的原因

「効果」フィールドを「何を実施したか」と解釈し、対策の実施内容を記載した。

### 根本原因

README の検証セクションに「効果: [同種の問題が再発していないか]」とフォーマット例があるが、対策実行直後に検証セクションを書く場合の「効果」の書き方が明示されていない。追記タイミングは「対応 Issue のクローズ時」と定義されており、クローズ時点でも効果が観測できるとは限らない。

## 対策

### 対策形式の検討

| 形式 | 案 | 評価 |
|------|-----|------|
| 行動規範 | 「効果は再発有無を確認してから記載すること」 | 低（再発リスク高） |
| 成果物要件 | README のフォーマット例に未確認時の記載パターンを追加 | 高 |

README の検証フォーマット例に、効果未確認時の記載パターンを追加する。

対策は小規模のため、本 PR 内で直接対応する（Issue 化不要）。

## 分類

- カテゴリ: 参照漏れ
- 失敗タイプ: 実行ギャップ
