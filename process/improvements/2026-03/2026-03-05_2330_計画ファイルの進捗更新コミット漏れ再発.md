# 計画ファイルの進捗更新コミット漏れ再発

## 事象

PR #1057（#1050）のマージ後、計画ファイル `prompts/plans/1050_split-workflow-instance-dto.md` の進捗チェックボックス更新（`[ ]` → `[x]`）がコミットされていないことをユーザーが発見した。

同種の問題は 3 回目の発生:
1. 2026-02-16: 初発（計画ファイル自体のコミット漏れ）
2. 2026-02-21: 再発（ルール再構成時に対策消失）
3. 2026-03-05: 本件（対策ステップが存在するが実行されなかった）

## 原因分析

### 直接原因

品質ゲート 6.4（`git status prompts/plans/`）のステップが実行されなかった。

### 根本原因

品質ゲート 6.4 は行動規範形式の対策（「確認する」）であり、構造的な強制がない。品質ゲートのチェックリストは項目数が多く、個別ステップのスキップが発生しやすい。

過去 2 回の対策はいずれも「フローにステップを追加する」形式で、実行を強制する仕組みがない。同じ形式の対策を 3 回適用して 3 回とも再発しており、行動規範形式の対策では解決できない問題であることが実証された。

## 対策

### 問題の性質: プロセス的

行動規範形式の対策が機能していないため、フロー組み込み形式に転換する。

### 対策案

lefthook の pre-commit で `prompts/plans/` に未コミットの変更がないか検証するスクリプトを追加する。PR ブランチで `prompts/plans/` 配下に unstaged/untracked の変更がある場合に警告を出す。

対策の Issue: #1063
