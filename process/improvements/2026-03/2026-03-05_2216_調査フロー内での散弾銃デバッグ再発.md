# 調査フロー内での散弾銃デバッグ再発

## 事象

PR #1040 の review-and-merge 中、Claude Rules Check の Group 2 が `error_max_turns` で失敗した。このエラーに対して:

1. Step 0（立ち止まる）の 3 問には回答した
2. しかし結論で「インフラ的な制限」と即断し、再実行を提案した
3. 原因未確認のまま先に進む傾向についてユーザーから指摘を受けた
4. 仮説追跡を開始し調査を進めたが、根本原因を確定できない段階で再実行を提案した
5. さらに「散弾銃デバッグが頻発するがどうするのがいいか」という問いに対して、改善記録を作成せずにいきなり分析と対策案を提示した
6. 改善記録の作成を先に行うべきとユーザーから指摘を受けた

問題は 3 層構造:
- 第 1 層: CI エラーに対して原因を検証せず「再実行すればいい」と即断
- 第 2 層: 指摘を受けて調査したが、仮説の検証が完了する前に再実行を提案
- 第 3 層: プロセス改善の文脈で、改善記録の作成をスキップして対策に飛んだ

## 原因分析

### 直接的原因

Step 0 の 3 問に回答した後、「対処は可逆か → 再実行は可逆」という判断が、「再実行で解決できる」という結論に短絡した。Step 0 は「対処に入る前に立ち止まる」ためのゲートだが、立ち止まった後の調査（Step 1〜6）を省略して「可逆だから問題ない」と判断した。

### 構造的原因

1. **Step 0 の限界**: Step 0 は「調査フローに入るかどうか」のゲートとして機能しているが、「調査フロー内で検証を省略する」パターンには介入しない。調査を開始した後に、途中で「もう十分」と判断して修正/対処に飛ぶ動きを止めるゲートがない

2. **「可逆性」が免罪符になっている**: Step 0 の問 3「対処は可逆か」が、本来の意図（不可逆な操作の防止）を超えて「可逆なら検証不要」という誤った推論を許容している

3. **繰り返しの本質**: 改善記録 4 件の系譜（2026-02-05 → 2026-02-20 → 2026-03-04 → 今回）が示すのは、各対策が「特定のトリガーポイントにゲートを追加する」アプローチであり、問題の性質（検証前に結論に飛ぶ思考パターン）自体に介入していないこと

4 件すべてが「即座の対策」カテゴリ、思考的問題。対策の蓄積にもかかわらず再発している事実は、これが AI エージェントの構造的限界である可能性を示唆する。

## 対策

### 対策形式の検証

これまでの対策はすべて「行動規範」または「成果物要件」形式。いずれも AI エージェントの推論過程での自己制御に依存しており、4 回再発している。

| 過去の対策 | 形式 | 再発 |
|-----------|------|------|
| problem-solving.md フレームワーク | 行動規範 | 再発 |
| 仮説検証ループ・仮説追跡表 | 成果物要件 | 再発 |
| Step 0（立ち止まる） | 成果物要件 | 再発 |

### 構造的限界の認識

同一カテゴリ（即座の対策）の改善記録が 4 件蓄積し、各対策後に再発している。これは以下の可能性を示す:

1. AI エージェントのコンテキスト窓内での注意力配分には構造的な限界がある
2. ルール・ゲートの追加は特定のトリガーポイントでは有効だが、推論過程全体での「検証前に結論に飛ぶ」パターンを完全には抑制できない
3. 対策の追加そのものがコンテキストを膨張させ、注意力を分散させている可能性がある

### 提案

この問題に対する追加のルール・ゲートの有効性は低い。ユーザーとの議論を経て方向性を決定する。

選択肢:
- A: 構造的限界として受容し、ユーザーが都度指摘する運用で対処する
- B: 既存のルールを整理・統合し、コンテキスト膨張を抑制する（ルールの量 vs 質のトレードオフ）
- C: 別のアプローチ（ユーザーとの議論で決定）

## 次のアクション

  - `settings.json` の hook 設定を実装する

## 既知手法との関連

| 本記録の要素 | 対応する既知手法 |
|------------|-----------------|
| 同一パターンの繰り返し改善 | Deming の PDCA サイクル — Check/Act で対策の有効性を評価し、無効なら方法自体を変える |
| 対策蓄積による効果逓減 | Alert fatigue（アラート疲れ）— セキュリティ分野で、アラートが多すぎると無視される現象 |
| AI の自己制御の限界 | Metacognition の限界 — 自己の思考プロセスを監視・制御する能力には上限がある |
