# Want と How の混同

## 発生日時

2026-02-06

## 概要

問題解決フレームワーク適用時に、手段（How）を Want として設定してしまった。

## 事象

`zoom-rhythm.md` の設計ブラッシュアップ完了基準から主観的表現を修正する際、Want を「ドキュメントの完了基準は客観的・検証可能であるべき」と設定した。

ユーザーから「客観的・検証可能であること、あるいは形式知化することは、『品質追求』という Want を叶える適切な手段にすぎない」と指摘を受けた。

## 根本原因

1. 直近の作業（完了基準の修正）に視野が狭まり、Want をプロジェクト理念レベルまで引き上げずに局所的な目標を Want に設定した
2. `problem-solving.md` に Want の設定基準（「プロジェクト理念レベルに置く」）が明記されていなかったため、ガードレールがなかった

## 対策

`problem-solving.md` に「Want の設定基準」セクションを追加:

- Want はプロジェクト理念レベル（品質の追求、学習効果の最大化）に置く
- How の具体例を表で示し、手段と目的の区別を明確化
- 手段が目的化するリスクを明記

## 学び

- 問題解決フレームワークの Want は、目の前のタスクの目標ではなく、プロジェクト理念に照らして設定する
- How を Want に昇格させると、Want に寄与しない活動を正当化するリスクがある
