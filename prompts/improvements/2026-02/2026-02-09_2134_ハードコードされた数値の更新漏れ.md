# ハードコードされた数値の更新漏れ

## 事象

Issue #326 でルールファイル `.claude/rules/rule-skill-writing.md` を新規追加した際、README.md に記載されたルールファイル数（「17 のルールファイル」）を更新しなかった。ユーザーの指摘により気づいて 18 に修正した。

## 原因分析

1. ルールファイルの追加は既存の「必須対応」パターン（データストア追加 → 設計書更新、API 追加 → OpenAPI 更新など）にカバーされていない
2. README にハードコードされた数値が存在することを認識していなかった
3. ドキュメント内のハードコードされた数値は、関連する変更時に自動的に検出されない構造的な問題がある

## 対策

根本原因はハードコードされた数値の存在そのものにある。対策の方向性は 2 つ:

1. **数値をハードコードしない**: 「複数のルールファイル」のように抽象的に記述する（根本解決）
2. **更新チェックを追加**: ルールファイル追加時に README を確認するルールを追加する（対症療法）

推奨: 1 の方向。ハードコードされた数値は変更のたびに手動同期が必要で、常に陳腐化のリスクがある。
