# 新規作業開始時の main 同期漏れ

## 事象

Issue #512 の作業開始時、ローカルの main が origin/main より1コミット遅れた状態で justfile を編集した。編集後に `git fetch` で差分に気づき、`git pull` で同期した。たまたまコンフリクトは発生しなかったが、コンフリクトした場合は編集のやり直しになっていた。

## 原因分析

### 表層的原因

Issue の内容確認後、ブランチ作成前に `git fetch` / `git pull` を実行しなかった。

### 根本原因

新規作業開始時に main の最新化を行うステップがプロセスに定義されていない。

既存プロセスとの比較:

| プロセス | main 同期のタイミング | 定義場所 |
|---------|---------------------|---------|
| PR 完了フロー Step 6 | Ready 前に `git fetch origin main` で差分確認 | CLAUDE.md |
| 新規作業開始時 | （なし） | — |

PR 完了時には同期確認があるが、作業開始時には同期確認がない。

## 対策

### 対策形式の検討

| 形式 | 案 | 評価 |
|------|-----|------|
| 行動規範 | 「作業前に pull すること」 | 低（忘れる） |
| フロー組み込み | Issue 駆動開発の手順にステップを追加 | 高 |

Issue 駆動開発の手順書に、ブランチ作成前の main 同期ステップを追加する。
