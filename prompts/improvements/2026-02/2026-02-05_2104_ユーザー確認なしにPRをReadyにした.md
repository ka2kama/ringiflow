# ユーザー確認なしに PR を Ready にした

## 事象

Issue #208 の実装完了後、ユーザーの確認を得ずに PR を Ready for Review に変更しようとした。

経緯:
1. 実装完了、`just check-all` 通過
2. 自己検証チェックリストを実施
3. ユーザーに確認せず `gh pr ready` を実行しようとした
4. ユーザーから「`/wrap-up` する前に ready にされてる」と指摘

## 原因分析

### 表層的な原因

ワークフローの理解不足。`/wrap-up` → ユーザー確認 → Ready の流れを認識していなかった。

### 根本原因

手順書には各ステップ（Draft 作成、Ready for Review チェックリスト）が個別に書かれているが、**ステップ間の依存関係と順序**が明文化されていない。

理想の PR 完了フロー:

```
実装完了 → 自己検証 → Draft PR 作成 → /wrap-up → ユーザー確認 → Ready
```

この流れが明示されていないため、AI エージェントは各ステップを個別に実行し、順序を無視してしまった。

## 対策

### 恒久対策

手順書に「PR 完了フロー」を明示的なチェックリストとして追加する。

Issue #242

## 次のアクション

- Issue #242 で手順書を更新
- Issue #241 は #242 に統合して close
