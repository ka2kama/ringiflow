# 廃止済み RFC 番号の採用

**発生日時**: 2026-02-06
**重大度**: 低
**カテゴリ**: 最新プラクティス

## 問題

2026年1月にプロジェクトを立ち上げた際、エラーレスポンス形式の参照として RFC 7807 を採用した。しかし RFC 7807 は 2023年7月に RFC 9457 により廃止（obsoleted）されており、プロジェクト開始時点で 2年半以上前に後継仕様が発行されていた。

18 ファイル・38 箇所に RFC 7807 の参照が残り、Issue #185 で一括更新することになった。

## 原因分析

### 直接原因

AI エージェント（Claude）がコード・ドキュメント生成時に、最新の RFC 9457 ではなく、より広く知られた RFC 7807 を使用した。

### 根本原因

**AI の訓練データにおける出現頻度バイアス**

RFC 7807（2016年発行）は Web 上の技術記事・チュートリアル・ライブラリのドキュメントに圧倒的に多く登場する。RFC 9457 は後方互換のため、既存記事の多くが更新されず旧番号のまま残っている。AI は訓練データ内の出現頻度に影響されるため、より一般的な旧名称を優先的に使用する。

### 適用漏れ

`.claude/rules/latest-practices.md` では依存関係のバージョン、言語のイディオム、ライブラリ API について最新プラクティスの採用を定めているが、**仕様の参照番号**はカバーされていなかった。

## 対策

### 1. latest-practices.md に「標準仕様の参照」を追加

`.claude/rules/latest-practices.md` に、RFC や W3C 勧告などの標準仕様を参照する際は最新の有効な番号を使用するルールを追加する。

### 2. プロジェクト初期の設計段階での確認

新プロジェクトの立ち上げ時や新しい標準仕様を参照する際に、「この仕様は最新か？後継仕様はないか？」を確認するプラクティスを意識する。

## 影響

- 実装への影響: なし（RFC 9457 は RFC 7807 と後方互換）
- 修正コスト: 低（機械的な文字列置換）
- 学習への影響: 中（最新仕様を参照していると思い込んでいた期間がある）
