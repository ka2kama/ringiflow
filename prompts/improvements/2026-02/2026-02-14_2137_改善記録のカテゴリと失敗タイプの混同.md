# 改善記録のカテゴリと失敗タイプの混同

## 事象

改善記録 `2026-02-14_2120_新規作業開始時のmain同期漏れ.md` で、カテゴリに「プロセスギャップ」（失敗タイプの値）を使用した。PR レビューで指摘を受け「知識-実行乖離」に修正されたが、同種のミスが再発している。

## 原因分析

### 表層的原因

分類を記載する際に、カテゴリの定義表（README.md の6種）を参照せず、問題の印象から値を選んだ。

### 根本原因

カテゴリと失敗タイプの定義が README に記載されているが、分類記載時に参照を強制する仕組みがない。README の定義表を毎回確認するかどうかは AI の判断に委ねられており、省略されやすい。

## 対策

### 対策形式の検討

| 形式 | 案 | 評価 |
|------|-----|------|
| 行動規範 | 「定義表を確認してから分類を記載すること」 | 低（再発リスク高。現状と変わらない） |
| 成果物要件 | README の定義済み値との照合を分類セクションに記載 | 高 |

改善記録の README に、分類記載時の照合手順を追加する。カテゴリの値が定義済み6種のいずれかであること、失敗タイプの値が定義済み3種のいずれかであることを照合する。
