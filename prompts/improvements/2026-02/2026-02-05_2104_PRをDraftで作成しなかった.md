# PR を Draft で作成しなかった

## 事象

Issue #208 の PR を作成する際、`gh pr create` を `--draft` オプションなしで実行した。

結果:
- PR が即座に Ready for Review 状態で作成された
- Claude Code Action の auto review が即座に実行された

## 原因分析

手順書（`docs/04_手順書/04_開発フロー/01_Issue駆動開発.md`）に `--draft` オプション付きのコマンド例が明記されているが、確認せずに PR を作成した。

## 対策

手順書に従う限り問題は発生しない。追加のルール化は不要。

手順書の該当箇所:

> ```bash
> gh pr create --draft --title "#34 Implement login feature" --body-file .github/pull_request_template.md
> ```

## 次のアクション

なし（手順書遵守で防止可能）
