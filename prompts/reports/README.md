# 診断レポート

プロジェクトの定点観測レポートを記録する。

## 目的

- ある時点でのプロジェクト状態のスナップショットを残す
- 時系列でのトレンド比較を可能にする
- 改善施策の効果測定の基準点を提供する

## ディレクトリ構造

ファイルを直接配置する（月別サブディレクトリは件数が増えた時点で検討）。

```
prompts/reports/
├── YYYY-MM-DD_HHMM_<トピック>.md
└── README.md
```

## ファイル命名規則

```
YYYY-MM-DD_HHMM_<トピック>.md
```

- `HHMM`: レポート生成時刻（24時間形式）
- `<トピック>`: レポートの種類（例: `プロジェクト健全性診断`）

## レポートの種類

| 種類 | 生成方法 | 頻度 |
|------|---------|------|
| プロジェクト健全性診断 | `/assess` スキル | 任意（月次推奨） |
| レトロスペクティブ | `/retro` スキル | 任意（週次推奨） |
