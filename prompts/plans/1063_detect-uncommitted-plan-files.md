# 1063: 計画ファイルのコミット漏れを lefthook pre-commit で検出する

## コンテキスト

### 目的
- Issue: #1063
- Want: 計画ファイルのコミット漏れを構造的に防止する（行動規範形式が 3 回失敗 → フロー組み込み形式へ転換）
- 完了基準:
  - lefthook pre-commit に `prompts/plans/` の未コミット変更を検出するスクリプトを追加
  - PR ブランチで `prompts/plans/` に unstaged/untracked の変更がある場合に警告を出す
  - 品質ゲート 6.4 のステップを更新（手動確認 → 自動検出への移行を反映）

### ブランチ / PR
- ブランチ: `feature/1063-detect-uncommitted-plan-files`
- PR: #1065（Draft）

### As-Is（探索結果の要約）
- `lefthook.yaml`: pre-commit に 8 hooks（`parallel: true`）。既存 hooks は `glob` + `{staged_files}` でステージ済みファイルを検査するパターン
- `scripts/check/`: bash スクリプトのパターン。引数ありなら指定ファイル、なしなら全ファイルを検査する二重モード
- 品質ゲート 6.4（`.claude/rules/dev-flow-issue.md`）: `git status prompts/plans/` の手動確認
- 今回の hook は既存パターンと異なる: ステージ済みファイルではなく「ワーキングツリーの未コミットファイル」を検出する
