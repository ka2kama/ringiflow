# セッションログのサニタイズ規則見直し

## コンテキスト

### 目的
- Want: セッションログが第三者に公開されても問題ない状態にする（セキュリティ/プライバシー保護）
- 完了基準:
  - セキュリティ/プライバシー系のサニタイズは維持される
  - ユーザー発言の内容（技術的指摘、フィードバック等）がセッションログに残る
  - 改善記録の直接引用禁止ルールは影響を受けない

### As-Is

現在のサニタイズ規則は 2 つの異なる関心事が混在している:

1. セキュリティ/プライバシー保護（認証情報、PII、内部 URL）→ Want に直結
2. 表現・トーンの校正（カジュアル表現、感情的発言）→ Want を超えている

結果として、ユーザー発言の大半が「技術的内容に該当しない」と判定され省略されている。3 月分のセッションログ 11 件中「議論の経緯」セクションが 0 件。

### 影響範囲

| ファイル | 変更内容 |
|---------|---------|
| `prompts/runs/README.md` | サニタイズ規則の見直し |

| `.claude/skills/wrap-up/SKILL.md` | 変更なし（参照先が変わるだけ） |

## 変更内容

### 1. `prompts/runs/README.md` サニタイズ規則の改定

現在の「表現・表記の校正」テーブルを廃止し、サニタイズの目的を明確化する。

変更前:
```markdown
## サニタイズ規則

セッションログは GitHub で公開されるため、以下の規則に従ってサニタイズする。

### 省略または伏せ字にする

| 対象 | 処理 |
|------|------|
| 認証情報（鍵、トークン、パスワード） | 完全に省略 |
| 個人を特定できる情報 | 省略または一般化 |
| 社内 URL、内部システム名 | 伏せ字（`[内部URL]`、`[システム名]`） |

### 表現・表記の校正

| 対象 | 処理 |
|------|------|
| カジュアルすぎる表現 | 丁寧語に整形 |
| 感情的・主観的な発言 | 技術的な内容に要約、または省略 |
| 誤字・脱字・表記揺れ | 修正・統一 |
```

変更後:
```markdown
## サニタイズ規則

セッションログは GitHub で公開されるため、第三者に見られても問題ない状態にする。

### 除去・伏せ字

| 対象 | 処理 |
|------|------|
| 認証情報（鍵、トークン、パスワード） | 完全に省略 |
| 個人を特定できる情報 | 省略または一般化 |
| 社内 URL、内部システム名 | 伏せ字（`[内部URL]`、`[システム名]`） |

### 表現・表記の校正

| 対象 | 処理 |
|------|------|
| カジュアルすぎる表現 | 丁寧語に整形 |
| 誤字・脱字・表記揺れ | 修正・統一 |

ユーザー発言は内容を保持する。口調の整形はするが、技術的に有用な指摘やフィードバックを省略しない。
```

- `scripts/check/sanitize-improvements.sh` — 改善記録のみを対象としており、影響なし
