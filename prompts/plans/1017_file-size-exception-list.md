# #1017 大型ファイルの早期検出: 件数ラチェットから例外リスト方式への移行

## 対象

- `scripts/check/file-size.sh`: 検出ロジックの全面書き換え
- `.config/file-size-exceptions.txt`: 新規作成（例外リスト）
- `.config/baselines.env`: `FILE_SIZE_MAX_COUNT` の削除

## 設計判断

### 1. pre-commit の挙動: 警告（exit 0）

Issue の完了基準が「警告」としている。`just check`（品質ゲート）で fail するため、pre-commit は早期通知の役割。

選択肢:
- A: 警告のみ（exit 0）— コミットはブロックしない ← 採用
- B: エラー（exit 1）— コミットをブロックする

理由: 開発中はファイルが一時的に超過することがある。品質ゲートが最終的なエンフォースメントポイント。

### 2. 例外リストの形式バリデーション

理由のないエントリを `file-size.sh` 実行時に検出して失敗させる。これにより例外追加時に理由の記載が強制される。

### 3. 陳腐化した例外の検出

例外リストにあるが 500 行以下のファイルを検出し、リストからの削除を提案する。ただし fail にはしない（情報提供のみ）。

### 4. pre-commit の検出対象

lefthook の `glob` で `.rs` / `.elm` ファイルに限定し、ステージされたファイルのみを検査する。テストファイル（`*/tests/*`、`*_test.rs`）は除外。
