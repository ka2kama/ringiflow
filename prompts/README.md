# prompts/

このディレクトリは、LLM（生成AI）に対するプロンプト（指示文）を「再現可能な設計資産」として残し、改善・再利用できる形で蓄積するための置き場です。

## 何を置くか

- タスク別テンプレ（実装、設計、レビュー、調査など）
- 定番の制約（出力形式、禁止事項、品質基準、テスト方針）
- few-shot（代表例：良い例/悪い例、要約版）
- 実行条件メモ（モデル、温度、ツール利用方針など）

## 置かない（置くなら必ず加工する）

- 秘密情報（鍵・トークン・社内URL・個人情報）
- 生ログの丸ごと保存

保存が必要な場合は、必ず以下を行ってください。
- 伏字・匿名化・要約（「汎用テンプレ + 匿名化した例」に落とす）
- 何が入力で、何が出力かを切り分ける（再利用できる単位にする）

## 推奨運用（最小）

- テンプレは [`prompts/templates/`](templates/) に置く
- 実際に使ったプロンプトは「テンプレのコピー + パラメータ差分」として残す（機密除去後）
  - 例: `prompts/runs/2026-01-12_add-xxx.md`
- PR/Issue に「使ったテンプレ」「主要なパラメータ」「変更理由」を短く紐づける

## 文体ガイドライン

runs に保存するプロンプトは、会話のやり取りそのままではなく、第三者が閲覧可能なフォーマルな日本語に整えること。

### 基本原則

- 敬体（です・ます調）または丁寧な依頼形（〜してください）を使用する
- 口語的な表現は避け、文語的な表現を選ぶ
- 誤字脱字、表記揺れを修正する

### 具体例

| 項目 | 修正前（口語） | 修正後（文語） |
|------|--------------|--------------|
| 動詞 | 〜して | 〜してください |
| 疑問 | 〜どうでしょうか | 〜どのような評価になりますか |
| 指示語 | この、あの | 本、当該 |
| 語彙 | つきあわせる | 照合する |
| 語彙 | 作り | 構成、構造 |
| 語彙 | いい感じに | 適切に |
| 委任 | 任せます | お任せします、ご判断ください |

### プロンプト記載例

```
# 修正前
ULIDとUUID v7の比較

# 修正後
ULID と UUID v7 を比較してください。
```

```
# 修正前
CLAUDE.mdの作りを全体的に見直して ultrathink

# 修正後
CLAUDE.md の構成を全体的に見直してください。 ultrathink
```
