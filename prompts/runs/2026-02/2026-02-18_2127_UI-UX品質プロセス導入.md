# UI/UX 品質プロセス導入

## 概要

Issue #445（デザイン・UI/UX の品質をベストプラクティス水準に引き上げる）に関連し、品質プロセス（ブラッシュアップループ、設計原則レンズ、品質チェックリスト）に UI/UX 観点を導入した。

## 実施内容

### 変更対象

| ファイル | 変更内容 |
|---------|---------|
| `.claude/rules/zoom-rhythm.md` | ギャップ発見の観点に UI/UX 行を追加（欠陥 3 + 品質向上 2） |
| `docs/60_手順書/04_開発フロー/02_TDD開発フロー.md` | 設計原則レンズに「UI/UX レンズ（フロントエンド変更時）」セクションを追加（毎 Refactor 3 + 完成時 4） |
| `docs/60_手順書/04_開発フロー/01_Issue駆動開発.md` | 品質チェックリストに「UI/UX 品質確認」セクション（7 項目）を追加 |

### 設計判断

1. UI/UX レンズの適用条件: フロントエンド変更時のみ（条件付き適用）
2. レンズの粒度: 既存のコード設計レンズと同じ（1 レンズ = 1 つの問い）
3. 品質チェックリストとの連動: 設計原則レンズと品質チェックリストの 1:1 対応を維持
4. 既知手法の参照: Nielsen's 10 Usability Heuristics（H1, H3, H4, H5, H8, H9）と WCAG 2.1 POUR 原則

### 既知手法の調査

methodology-design.md の方針に従い、既知手法を地図として把握した上で設計した。

参照した既知手法:
- Nielsen's 10 Usability Heuristics (1994, 2024 更新) — UI/UX レンズの設計基盤
- WCAG 2.1 POUR 原則 (W3C, 2018) — アクセシビリティレンズの根拠
- ゲシュタルト心理学の UI 適用 (Wertheimer, 1923 → NN/g) — 視覚的階層レンズの根拠

## 判断ログ

- 設計: ブラッシュアップループのギャップ発見の観点（zoom-rhythm.md）は「代表的な観点であり、網羅的ではない」という既存設計に沿って UI/UX 行を追加。テーブル構造の変更なし
- 設計: 設計原則レンズは既存のコードレンズ（毎 Refactor / モジュール完成時）と並列に「毎 Refactor / ページ・コンポーネント完成時」を追加。タイミングの粒度を UI 作業に合わせて調整
- 収束確認: 検証で品質チェックリストに「エラー表示」「破壊的操作の防御」の 2 項目が欠落していることを発見し、追加修正
