# デザインガイドラインを styles.css から詳細設計書に分離

Issue: #661

## 概要

`styles.css` に埋め込まれていたデザインガイドライン（約200行の CSS コメント）を `docs/40_詳細設計書/13_デザインガイドライン.md` に分離した。CLAUDE.md の原則「コード内のコメントは簡潔に、詳細解説はナレッジベースに書いてコードからリンクする」に基づく対応。

## 実施内容

### Phase 1: デザインガイドライン Markdown ドキュメント作成

`docs/40_詳細設計書/13_デザインガイドライン.md` を新規作成。`styles.css` のコメント内容をフォーマット変換のみで Markdown 化した。

構成: デザインガイドライン（コンポーネント別トークン定義）、デザイン原則、スペーシング、レイアウト構成、色の使い分け、レスポンシブ。

### Phase 2: styles.css コメント簡潔化

`styles.css` を 317行から 126行に簡潔化。約200行のガイドラインコメントを9行のヘッダコメント（概要 + 新ドキュメントへのリンク）に置き換えた。

### Phase 3: 参照先の更新

品質プロセス関連ファイル 4つ、計 8箇所の参照先を更新した。

- `.claude/rules/zoom-rhythm.md`: デザイントークン乖離の観点
- `docs/60_手順書/04_開発フロー/01_Issue駆動開発.md`: 品質チェックリスト 2箇所
- `docs/60_手順書/04_開発フロー/02_TDD開発フロー.md`: 参照・チェック項目 3箇所
- `docs/80_ナレッジベース/methodology/独自フレームワークと既知手法の対応.md`: 参照 2箇所

## 判断ログ

- 「デザイントークン」（`styles.css` の `@theme` で定義される CSS カスタムプロパティ）と「デザインガイドライン」（トークンの使い方のルール）を区別し、参照先を分離した。`02_TDD開発フロー.md` の1箇所は元々両方を一つの参照で案内していたため、2つの参照に分離した
- `prompts/plans/` 内の過去の計画ファイルと `docs/70_ADR/027_Tailwind_CSS導入.md` は、それぞれ作成時点の記録・トークン定義の参照であり、更新対象外と判断した
