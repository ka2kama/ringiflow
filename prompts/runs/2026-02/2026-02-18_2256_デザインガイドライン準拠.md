# 既存ビューのデザインガイドライン準拠

Issue: #670 | PR: #671 | 計画: `prompts/plans/670_design-guideline-compliance.md`

## 概要

Epic #445 で策定したデザインガイドライン（`docs/03_詳細設計書/13_デザインガイドライン.md`）に対し、策定前に書かれた既存ビューの乖離を一括修正した。全変更は CSS クラスの修正・コンポーネント置換であり、ロジック変更なし。

## 実施内容

### Phase 1: 散在する小修正

- パンくずリストの統一（7ファイル）: 2パターン混在を `mb-4 flex items-center gap-2 text-sm` + `/` セパレータに統一
- バッジ色パターンの統一（3ファイル + テスト1ファイル）: `bg-{color}-50 text-{color}-600 border-{color}-200` に統一
- フォーカススタイルの補完（1ファイル）: Workflow/List.elm のステータスドロップダウンに `focus-visible:ring-2` を追加
- スペーシングの修正（1ファイル）: Home.elm の余分な `mt-4` を除去

### Phase 2: 詳細ページ構造の統一

- セクションカード化: Workflow/Detail.elm（4関数）と Task/Detail.elm（2関数）の `div []` を `div [ class "rounded-lg border border-secondary-200 bg-white p-6 shadow-sm" ]` に変更
- インラインスピナー排除: Workflow/Detail.elm の手書きスピナー2箇所を `LoadingSpinner.view` に置換

## 判断ログ

- パンくずの統一パターンとして `mb-4`（ガイドライン定義値）+ flex レイアウト + `/` セパレータ（Web 標準）を選択
- Secondary バッジの bg を `-100` に維持（ガイドラインの中性ステータス記載に従う）、text のみ `-600` に統一
- `viewSteps` はセクションカード化の対象外とした（個別ステップが既にカード化されており、入れ子カードの複雑さを避ける）
- `Data/AuditLog.elm` の `resultToCssClass`（未使用デッドコード）は本 Issue のスコープ外とした
