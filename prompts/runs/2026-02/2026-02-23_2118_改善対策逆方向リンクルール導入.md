# 改善対策逆方向リンクルール導入

## 概要

- Issue: #767
- PR: #803
- ブランチ: `feature/767-improvement-measure-backlink`

## 変更内容

改善対策をドキュメントに配置する際に逆方向リンク（HTML コメント）を追加するルールを導入した。

### 背景

PR #589 で CLAUDE.md に追加した改善対策が、ドキュメント再構成時に消失した（PR #766 で発覚）。改善対策がドキュメントの通常の文章と区別できないことが根本原因。

### 変更ファイル

| ファイル | 変更内容 |
|---------|---------|
| `process/improvements/README.md` | 「対策配置時の逆方向リンク（必須）」セクションを追加 |
| `docs/60_手順書/04_開発フロー/01_Issue駆動開発.md` | 品質ゲートフロー 6.4 に逆方向リンクを追加 |

### 逆方向リンクの書式

```markdown
<!-- 改善: process/improvements/YYYY-MM/ファイル名.md -->
対策として追加された記述
```

## 判断ログ

特筆すべき設計判断なし。改善記録で既に定義された HTML コメント形式をそのまま採用。
