# Epic #841 完了と rust-script テスト自動化

## 概要

Epic #841（シェルスクリプトの rust-script 移行）の全サブ Issue 完了を確認しクローズした。その過程で rust-script のユニットテスト（全 67 件）が `just check` / CI で自動実行されていない問題を発見し、Issue #862 として対応・マージした。

## 実施内容

### 1. Epic #841 クローズ

4 つのサブ Issue がすべてマージ済みであることを確認:

| Issue | 対象スクリプト | PR |
|-------|--------------|-----|
| #834 | `instrumentation.sh` | #847 |
| #835 | `improvement-records.sh` | #852 |
| #836 | `impl-docs.sh` | #857 |
| #837 | `sync-epic.sh` | #861 |

ADR-056 のスコープ拡張（`scripts/check/` → `scripts/`）も反映済み。Issue チェックボックスを更新し、Epic をクローズした。

### 2. rust-script テスト自動化ギャップの発見

rust-script ファイルは Cargo workspace 外の独立スクリプトであるため、`cargo test` では実行されない。`rust-script --test <file>` で明示的に呼び出す必要があるが、`justfile` にも `parallel.sh` にも CI にもその定義がなかった。結果として 67 件のユニットテストが存在するのに一度も自動実行されない状態だった。

### 3. Issue #862 実装（PR #863）

justfile に `test-rust-scripts` レシピを追加し、`test` 集約レシピと `parallel.sh` の Non-Rust レーンに組み込んだ。

変更ファイル:
- `justfile`: `test-rust-scripts` レシピ追加、`test` 集約に追加
- `scripts/check/parallel.sh`: Non-Rust レーンに `just test-rust-scripts` 追加

## 判断ログ

- `test-rust-scripts` を Non-Rust レーンに配置した（rust-script は Cargo ビルドキャッシュを使用しないため、Rust レーンと並列実行してもロック競合しない）
- 各スクリプトファイルを個別に `rust-script --test` で呼び出す方式を採用（glob パターンでの一括実行は rust-script がサポートしていないため）
