# レトロ Issue 対象期間算出ロジック修正

## 概要

週次レトロスペクティブ Issue 自動生成ワークフロー（`weekly-retro.yaml`）の対象期間算出ロジックを、固定の「前週月曜〜日曜」ウィンドウから、前回レトロレポートの日付〜今日に変更した。

## 実施内容

### ワークフロー修正（`.github/workflows/weekly-retro.yaml`）

1. `permissions` に `contents: read` を追加（`actions/checkout` がリポジトリを読むために必要）
2. `actions/checkout@v6` ステップを追加（sparse-checkout で `prompts/reports` のみ取得）
3. 対象期間算出ロジックを前回レトロ基準に変更（フォールバック: 7日前）
4. step id を `week` → `period` にリネームし、参照箇所も更新

### 設計ポイント

- `/retro` スキルの期間算出方式と統一。スキルは `prompts/reports/` 内の最新レトロレポートファイル名から日付を取得する方式
- sparse-checkout で `prompts/reports` のみ取得し、クローン量を最小化。cone mode（ディレクトリ単位指定に最適化）を使用
- GitHub Actions で `permissions` を明示指定すると未指定の権限は `none` になるため、`contents: read` の追加が必須

## 判断ログ

- 特筆すべき判断なし（`/retro` スキルの既存方式に統一する方針が明確だったため）
