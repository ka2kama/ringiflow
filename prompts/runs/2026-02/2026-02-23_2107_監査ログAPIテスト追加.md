## 判断ログ

- DynamoDB のシードデータなし制約に対し、テスト内でロール作成操作を行い監査ログを生成する方式を採用。他の API テスト（`dashboard/stats.hurl` 等）と同じパターン
- アサーション方針: `actor_id`、`actor_name`、`action`、`result`、`resource_type` は決定的な値のため `==`。`id`、`resource_id` は UUID 自動生成のため `exists`。`created_at` はタイムスタンプのため `matches` パターン
