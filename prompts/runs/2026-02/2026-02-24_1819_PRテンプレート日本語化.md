# PR テンプレートのセクション名日本語化

Issue: #897
PR: #898

## 概要

PR テンプレートの英語セクション名を日本語化した。AI が PR 本文を英語で書いてしまう再発問題（改善記録あり）への構造的対策。直訳ではなく、セクションの実体に即した命名とした。

## 実施内容

### 改善記録の作成

- `process/improvements/2026-02/2026-02-24_1818_PR本文を英語で記述した.md` を作成
- Issue #897 を作成

### PR テンプレートのセクション名変更

初回コミットでは直訳（`Self-review` → `セルフレビュー`、`Test plan` → `テスト計画`）で全参照を更新。直訳的な命名を避けるようフィードバックを受け、内容に即した命名に改善した。

最終的な変換:

| 英語 | 直訳（中間） | 最終 | 理由 |
|------|------------|------|------|
| Summary | 概要 | 概要 | 自然 |
| Self-review | セルフレビュー | 品質確認 | 実体は品質チェックリストの確認結果 |
| Test plan | テスト計画 | 確認項目 | 実体はチェックリスト形式の確認事項 |

### 参照の更新

PR テンプレートのセクション名を参照する全ファイルを更新。retro/assess スキルの「セルフレビュー」や設計書の「テスト計画」は別文脈のため変更対象外と判別した。

更新対象:

- `.github/pull_request_template.md`
- `.claude/rules/zoom-rhythm.md`
- `docs/04_手順書/04_開発フロー/01_Issue駆動開発.md`
- `docs/04_手順書/02_プロジェクト構築/03_GitHub設定.md`
- `docs/06_ナレッジベース/methodology/ISO25010.md`
- `docs/06_ナレッジベース/methodology/独自フレームワークと既知手法の対応.md`

### 改善記録の修正

lint で検出されたカテゴリ・失敗タイプの不正値を修正:

- カテゴリ: `ドキュメント規約違反` → `知識-実行乖離`
- 失敗タイプ: `ルール遵守漏れ` → `プロセスギャップ`

## 判断ログ

- 命名方針: 直訳ではなく実体に即した命名を採用。ユーザーのフィードバックにより、「セルフレビュー」（カタカナ転写）→「品質確認」（内容を表す）、「テスト計画」（やや大げさ）→「確認項目」（実体はチェックリスト）に変更
- 更新スコープの判別: `セルフレビュー` / `テスト計画` は PR テンプレート文脈の参照のみ更新。retro/assess スキルの「セルフレビュー」（分析の自己レビュー機能）や設計書の「テスト計画」（機能テスト設計）は別の意味なので変更しない
