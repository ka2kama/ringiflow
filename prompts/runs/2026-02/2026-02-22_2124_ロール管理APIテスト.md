## 概要

ロール管理 CRUD 5 エンドポイントの Hurl API テストを追加し、API テストカバレッジギャップを解消した。先行 Story #776（ユーザー管理 API テスト）のパターンを踏襲。

## 実施内容

### テストファイルの作成

`tests/api/hurl/role/` ディレクトリに 5 テストファイルを作成:

| ファイル | エンドポイント | テスト内容 |
|---------|-------------|-----------|
| `list_roles.hurl` | GET /api/v1/roles | seed データの tenant_admin, user を検証 |
| `create_role.hurl` | POST /api/v1/roles | 正常系（201）+ 重複名（409） |
| `get_role.hurl` | GET /api/v1/roles/{role_id} | 正常系（200）+ 存在しない ID（404） |
| `update_role.hurl` | PATCH /api/v1/roles/{role_id} | 名前・説明・権限の変更（200） |
| `delete_role.hurl` | DELETE /api/v1/roles/{role_id} | 正常系（204）+ 存在しない ID（404） |

### アサーション方針

API テストルール（`.claude/rules/api-test.md`）に従い、値の決定性で手段を選択:

- `==`: seed データの ID・名前、リクエストボディの入力値、ドメインロジックの不変条件（`is_system == false`）
- `matches`: タイムスタンプ（`created_at`, `updated_at`）
- `count`: 配列型フィールド（`permissions`）
- `>=`: 他テストの実行結果に依存しうる値（seed データの `user_count`）
