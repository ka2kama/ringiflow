# 見出し階層とタイポグラフィの統一

## 概要

Issue #612（Epic #445 Story 2）として、フロントエンドの見出し階層とタイポグラフィを統一した。HTML セマンティクスの正規化（h2 → h1）、セクション見出しマージンの統一（mb-3 → mb-4）、スタイルの不整合修正を実施。

## 実施内容

### Issue 精査

全 17 ページの見出しパターンを実測し、3 つの問題を特定した:

1. HTML 要素の不統一: ページタイトルに h1（4 ページ）と h2（8 ページ）が混在
2. マージンのばらつき: セクション見出しで mb-3（9 箇所）と mb-4（4 箇所）が混在
3. セマンティクスとスタイルの不一致: Workflow/Detail「進行状況」が h2 にサブセクションスタイル（`text-sm`）を使用

例外パターン（404 ページ、成功メッセージ）は意図的な設計と判断し、対象外とした。

### Phase 1: ページタイトル正規化

8 ページの h2 → h1 変更を実施。SPA のメインコンテンツ領域では h1 が HTML セマンティクスの標準。

対象: Home, User/New, User/List, User/Edit, Role/New, Role/List, Role/Edit（2 箇所）, AuditLog/List

Workflow/New.elm には mb-6 が欠けていたため追加。他のページは親 div の mb-6 または space-y-6 コンテナで間隔を確保済み。

### Phase 2: セクション見出しマージン統一 + スタイル修正

- mb-3 → mb-4 の統一（Workflow/Detail 5 箇所、Task/Detail 3 箇所、Role/List 1 箇所）
- Workflow/Detail「進行状況」のスタイルを `text-sm font-semibold text-secondary-700` → `text-lg font-semibold text-secondary-900` に変更（h2 のまま維持し WCAG 見出しレベルスキップ禁止に準拠）
- CSS クラスの順序正規化（Role/List、User/Detail）
- styles.css のタイポグラフィガイドラインにマージン規約を追記

## 判断ログ

- h1 統一: SPA メインコンテンツ領域では h1 が HTML セマンティクス標準。ConfirmDialog の h2 はダイアログ内コンテキストのため変更不要
- mb-4 選択: 1rem（16px）は 4px グリッドの 4 単位で業界標準のスペーシング。mb-3（12px）より変更量が多いが品質を優先
- 「進行状況」対応: h3 に変更すると h2 レベルをスキップし WCAG 違反。h2 のまま他のセクション見出しとスタイルを統一
