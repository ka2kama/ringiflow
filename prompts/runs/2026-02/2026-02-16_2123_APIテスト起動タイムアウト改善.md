# API テスト起動タイムアウト改善

## 概要

Issue #552 に対応し、API テスト・E2E テストのサービス起動スクリプトにビルドフェーズを追加した。`cargo run` の前に `cargo build` を実行することで、コンパイル時間をヘルスチェックタイムアウトから分離する。

## 実施内容

- `scripts/run-api-tests.sh`: `cargo build` で3サービスを事前ビルドしてから `cargo run` で起動する構成に変更
- `scripts/run-e2e-tests.sh`: 同一の根本原因（`cargo run` がコンパイル時間を含む）を持つため、同様の修正を適用

## 判断ログ

特筆すべき判断なし。Issue の提案通りの実装で、技術的な選択肢の検討は不要だった。

- 変更ファイル: `scripts/run-api-tests.sh`, `scripts/run-e2e-tests.sh`
