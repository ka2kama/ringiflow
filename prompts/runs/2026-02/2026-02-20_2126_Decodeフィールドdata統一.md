# Decode.field "data" インライン使用の Data モジュールへの統一

Issue: #718
PR: #721

## 概要

`Api/Workflow.elm` 内で `Decode.field "data"` をインライン使用していた 8 箇所を、`Data` モジュールの `detailDecoder` に統一するリファクタリングを実施した。`frontend.md` に追加済みのデコーダ命名規約（`decoder` / `detailDecoder` / `listDecoder`）に準拠させた。

## 実施内容

### Phase 1: Data モジュールに `detailDecoder` を追加

- `Data/WorkflowInstance.elm` に `detailDecoder` を追加（`Decode.field "data" decoder`）
- `Data/WorkflowComment.elm` に `detailDecoder` を追加（同上）
- 両モジュールのテストファイルに `detailDecoder` のユニットテストを追加（正常系 + data フィールド欠損時のエラー）

### Phase 2: `Api/Workflow.elm` のインライン使用を置換

- 8 箇所の `Decode.field "data" WorkflowInstance.decoder` / `WorkflowComment.decoder` を `WorkflowInstance.detailDecoder` / `WorkflowComment.detailDecoder` に置換
- Refactor: 置換後にただのエイリアスとなった `createResponseDecoder` / `submitResponseDecoder` を削除し、呼び出し元から `WorkflowInstance.detailDecoder` を直接使用
- 不要になった `Json.Decode` import を削除

## 判断ログ

- Refactor: `createResponseDecoder` / `submitResponseDecoder` が置換後に `WorkflowInstance.detailDecoder` のエイリアスになったため削除した。間接参照を除去し、コードの簡潔さを向上
- スコープ判断: `Api/Auth.elm` の `userDecoder` も `Decode.field "data"` をインライン使用しているが、`User` 型が `Shared` モジュール（Data モジュールではない）に定義されているため、本 Issue のスコープ外とした
