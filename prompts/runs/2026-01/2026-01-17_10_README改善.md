# README.md をポートフォリオ向けに改善

## 概要

プロジェクトルートの README.md を第三者に技術力とプロジェクトへの姿勢が伝わるよう全面改訂した。

## 背景と目的

既存の README.md は基本情報のみで、第三者が見た際にプロジェクトの価値や技術的な取り組みが十分に伝わらなかった。ポートフォリオとして機能するよう、以下を実現する必要があった。

- 技術選定の意図を示す
- プロジェクト理念（学習効果、品質追求）を明示する
- AI 主導の開発という実験的な側面をアピールする
- ドキュメント体系の充実度を見せる

## 実施内容

### 追加したセクション

1. **バッジ**: CI ステータス、Rust/Elm バージョン、ライセンス（CC0-1.0）
2. **プロジェクト理念**: 学習効果の最大化、品質の追求
3. **技術スタック**: 選定理由カラムを追加
4. **アーキテクチャ図**: Mermaid で全体構成を可視化
5. **設計パターン**: BFF、CQRS、マルチテナントの目的を表形式で説明
6. **ドキュメント体系**: 「知りたいこと→参照先」形式に変更
7. **開発フロー**: Issue 駆動開発、AI 主導の開発
8. **ディレクトリ構成**: プロジェクト構造を俯瞰

### 調整したセクション

- 冒頭説明: 「AI エージェント主導で開発し、オーナーは主に指示・レビューを行う実験」であることを明記
- Getting Started: ドキュメント体系と重複していた部分を簡略化
- ライセンス: セクション削除（バッジで CC0-1.0 を表示）

## 成果物

- `README.md`: 全面改訂

## ユーザープロンプト（抜粋）

> プロジェクトルートの README をもっといい感じにしたいです。

> README の改善で特に重視したい点はどれですか？
> → ここは多分、自分がというより第三者がみるもの。その観点からだと？

> CLAUDE.md にあるもののうち、README にもあるとよさそうなものは調整してください。

> 主に AI エージェントに作らせる（オーナーはほぼ指示するだけ）という意味では実験的な意味合いもかねています。

## 学んだこと

- README はプロジェクトの「顔」であり、第三者にとっての第一印象を決める
- 技術選定の「理由」を書くことで、単なる技術リストではなく思考プロセスが伝わる
- AI 主導の開発という実験的な側面は、ユニークさとしてアピールできる
- ライセンスは目的に応じて選ぶ。学習プロジェクトで自由に使ってほしい場合は CC0 が適切
