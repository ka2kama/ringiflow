# 2026-01-14_05: ドキュメントリンク整備

## 概要

ドキュメント内でバッククォートのみで記載されていたファイル参照を、Markdown リンク形式に変換した。
GitHub や IDE 上でクリックして直接参照先に移動できるようになった。

## 背景と目的

プロジェクトの「暗黙知ゼロ」理念に基づき、ドキュメント間のナビゲーションを改善する。
バッククォートで囲まれたパス（例: `` `docs/01_要件定義書/` ``）は視覚的にファイルパスと認識できるが、クリックしても移動できない。
Markdown リンク化することで、ドキュメント間の移動が容易になり、可読性と利便性が向上する。

## 実施内容

1. Explore エージェントでリポジトリ内のファイル参照を網羅的に調査
2. 各ファイルのバッククォート参照を Markdown リンク形式に変換
3. 相対パスを適切に設定（同一ディレクトリ内、親ディレクトリへの参照など）

### 変換例

**変換前**:
```markdown
| 何を作るか（WHAT） | `docs/01_要件定義書/` | `00_はじめに.md` |
```

**変換後**:
```markdown
| 何を作るか（WHAT） | [`docs/01_要件定義書/`](docs/01_要件定義書/) | [`00_はじめに.md`](docs/01_要件定義書/00_はじめに.md) |
```

## 学んだこと

- Markdown リンクでは相対パスが使用可能。ファイルの配置場所に応じて `../` で親ディレクトリを参照できる
- テーブル内でもリンクは正常に機能する
- バッククォートとリンクを組み合わせる場合は `[`\`text\``](path)` の形式で記述する
