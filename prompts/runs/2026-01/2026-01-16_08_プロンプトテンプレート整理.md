# プロンプトテンプレート整理

## 概要

使用していないプロンプトテンプレートを削除し、セッション引き継ぎ方針を明確化した。

## 背景

- `prompts/templates/` に汎用テンプレートがあったが、実際には使用していなかった
- セッション引き継ぎ用のテンプレートを作成しようとしたが、GitHub Project Board があれば不要と判断

## 実施内容

### 削除したファイル

| ファイル | 理由 |
|---------|------|
| `prompts/templates/01_実装依頼.md` | 未使用 |
| `prompts/templates/02_ドキュメント作成.md` | 未使用 |

### 更新したファイル

| ファイル | 内容 |
|---------|------|
| `prompts/README.md` | テンプレート参照を削除、セッション引き継ぎ方針を追記 |

## 設計判断

### セッション引き継ぎに専用ファイルは不要

検討した選択肢:

| 案 | 評価 |
|----|------|
| `prompts/templates/` にテンプレート | 二重管理になる |
| `prompts/handoffs/` に引き継ぎファイル | 古くなるリスク |
| `prompts/current.md` に現在地を記録 | 更新忘れのリスク |
| Project Board で管理 | 採用。単一の情報源 |

結論: Project Board の Todo 列を確認すれば次のタスクが分かる。

引き継ぎプロンプトは以下で十分:

```
Project Board を確認して、次のタスクを進めてください。
```

### プロンプトテンプレートが不要な理由

Claude Code での対話では、細かいテンプレートを埋めるより自然な会話の方が効率的。
必要に応じて都度追加する方針とする。

## 学んだこと

- 二重管理は避ける。情報源は一つに
- 「あったら便利かも」で作ったものは使われない
- 実際に必要になってから作る（YAGNI）
