## 設計判断

### RemoteData の Failure から ApiError を削除

elm-review の警告「未使用の型引数」に対応するため、`Failure ApiError` を `Failure` に変更。

理由:
- 現在の UI ではエラーの種類によらず同じメッセージを表示している
- エラー詳細が必要になった場合は再度追加可能

### NotAsked の削除（List.elm）

一覧ページでは `init` 時に必ず API を呼び出すため、`NotAsked` 状態は不要。
`Loading` から開始するシンプルな状態遷移にした。
