# ApiResponse エンベロープパターン廃止

## セッション情報

- 日時: 2026-03-09
- Issue: #1095
- PR: #1097
- ブランチ: `feature/1095-remove-api-response-envelope`

## 概要

ADR-065 の決定に基づき、`ApiResponse<T>` エンベロープパターン（`{"data": T}`）を全レイヤーから除去した。84 ファイル、約 1,800 行の純減。

## 実施内容

### Phase 1: Core Service ハンドラ

- 9 ファイル・66 箇所の `ApiResponse::new(data)` → `data` に置換
- ハンドラテストから `ApiResponse` の参照を除去

### Phase 2+3+6: BFF クライアント・ハンドラ・OpenAPI

- BFF クライアント: `handle_response<T>` が `ApiResponse<T>` ではなく `T` を直接デシリアライズするように変更
- BFF ハンドラ: `.data` アクセスと `ApiResponse::new()` を除去
- OpenAPI アノテーション: `body = ApiResponse<T>` → `body = T`

### Phase 4: PaginatedResponse リネーム

- `PaginatedResponse` の `data` フィールドを `items` にリネーム
- バックエンド（`paginated_response.rs`, `audit_log.rs`）、フロントエンド（`Data/AuditLog.elm`, `Page/AuditLog/List.elm`）、テスト（`Data/AuditLogTest.elm`, Hurl）を同時更新

### Phase 5: フロントエンド

- `Api.elm` の `expectJson` から `Decode.field "data"` を除去
- `getRaw` / `expectJsonRaw` を削除（`expectJson` と同一になったため）
- `Api/AuditLog.elm` の `Api.getRaw` → `Api.get` に変更

### Phase 7: Hurl API テスト

- Perl one-liner で 37 ファイル・627 行の `$.data.` → `$.`、`$.data[` → `$[`、`$.data"` → `$"` を機械的置換

### Phase 8: ApiResponse 型削除・OpenAPI 再生成

- `api_response.rs` を削除、`shared/src/lib.rs` から export を除去
- `just openapi-generate` で OpenAPI 仕様を再生成（`ApiResponse_*` スキーマが除去された）
- insta スナップショット更新

## 判断ログ

### PaginatedResponse.data → items のリネーム

ApiResponse の `data` フィールド除去に合わせて、`PaginatedResponse` の `data` フィールドも `items` にリネームした。`data` という名前はエンベロープパターンの名残であり、`items` の方がページネーション結果のリスト内容を正確に表現する。

### Phase の統合

Phase 2（BFF クライアント）、Phase 3（BFF ハンドラ）、Phase 6（OpenAPI アノテーション）は密結合しているため、1 コミットに統合した。BFF クライアントの戻り値型変更がハンドラと OpenAPI アノテーションに連鎖するため、分離するとコンパイルが通らない。
