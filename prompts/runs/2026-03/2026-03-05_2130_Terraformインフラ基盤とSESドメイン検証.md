# Terraform インフラ基盤と SES ドメイン検証

- Issue: #1041
- PR: #1042
- ブランチ: `feature/1041-terraform-infrastructure`

## 概要

Terraform インフラ基盤（provider、state backend、環境別設定）を整備し、SES ドメイン検証モジュールを作成した。#879（SES 連携）で Terraform 基盤が未整備であることが判明し、分離された Issue。

## 変更内容

### Phase 1: Terraform 基盤設定

- `infra/terraform/environments/dev/versions.tf` — Terraform >= 1.0、AWS provider ~> 5.0
- `infra/terraform/environments/dev/backend.tf` — S3 + DynamoDB state backend
- `infra/terraform/environments/dev/providers.tf` — AWS provider（default_tags 付き）
- `infra/terraform/environments/dev/variables.tf` — 共通変数（aws_region、domain_name）
- `infra/terraform/environments/dev/terraform.tfvars.example` — 変数テンプレート
- `.gitkeep` ファイルの削除（dev/stg/prod/modules/network）

### Phase 2: SES ドメイン検証モジュール

- `infra/terraform/modules/ses/main.tf` — `aws_sesv2_email_identity`（DKIM 付き）+ `aws_sesv2_configuration_set`
- `infra/terraform/modules/ses/variables.tf` — domain_name、environment
- `infra/terraform/modules/ses/outputs.tf` — DKIM トークン、configuration_set_name
- `infra/terraform/environments/dev/main.tf` — SES モジュール呼び出し
- `infra/terraform/environments/dev/outputs.tf` — DKIM トークンの出力

### Phase 3: SES メール送信テスト手順

- `docs/60_手順書/03_運用/02_SESメール送信テスト.md` — Terraform apply、DKIM 設定、テストメール送信、トラブルシューティング

## 判断ログ

### State backend のブートストラップ

S3 バケットと DynamoDB テーブルは手動作成を前提とした。ブートストラップスクリプトは一度だけの操作に対してオーバーエンジニアリングのため見送り。既存の手順書（`docs/60_手順書/02_プロジェクト構築/02_Terraform基盤構築.md`）に作成手順が記載済み。

### dev 環境のみ実装

YAGNI に基づき、現時点で必要な dev 環境のみ実装。stg/prod は将来の Issue で対応。

### SES v2 API の採用

旧 SES v1（`aws_ses_domain_identity`）ではなく SESv2（`aws_sesv2_email_identity`）を使用。SESv2 はドメイン検証時に自動的に DKIM を設定し、`aws_sesv2_configuration_set` でメトリクス追跡が可能。

### Terraform バージョン制約

`>= 1.0` と広い互換性を確保。基盤設定では最新機能は不要であり、チームメンバーの環境差異を許容する方針。

## 制約事項

- ローカル環境に Terraform がインストールされていないため、`terraform init/validate` はローカルでは未検証
- 実際の `terraform apply` は AWS アカウント設定後に手動で実施する
