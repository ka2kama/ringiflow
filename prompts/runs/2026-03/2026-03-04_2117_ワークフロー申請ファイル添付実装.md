# ワークフロー申請ファイル添付実装

## 概要

Issue #884 のフルスタック実装。ワークフロー定義の `file` フィールド型追加（バックエンド）、FileUpload コンポーネント（D&D + ファイル選択 + 進捗バー）、申請フォームへの統合、詳細画面での添付ファイル表示・ダウンロードを 5 Phase で実装した。

PR: #1031

## 実施内容

### Phase 1: バックエンド — definition_validator に file フィールド型追加

- `definition_validator.rs` に `validate_file_field` を追加
- `FileConfig` のバリデーション: `maxFiles`（1-100）、`maxFileSize`（1B-100MB）、`allowedTypes`（`FileValidation::ALLOWED_CONTENT_TYPES` のサブセット）
- `valid_types` に `"file"` を追加
- テスト 10 件追加

### Phase 2: フロントエンド — Data/Document.elm + Api/Document.elm + FormField 拡張

- `Data/Document.elm`: `Document`, `UploadUrlResponse`, `DownloadUrlResponse` 型とデコーダー
- `Api/Document.elm`: `requestUploadUrl`, `uploadToS3`, `confirmUpload`, `listWorkflowAttachments`, `requestDownloadUrl` API 関数
- `Data/FormField.elm`: `FieldType` に `File FileConfig` バリアント追加、`FileConfig` 型（`maxFiles`, `maxFileSize`, `allowedTypes`）とデフォルト値
- テスト: `DocumentTest.elm`（デコーダー 13 件）、`FormFieldTest.elm`（FileConfig 4 件）

### Phase 3: フロントエンド — Component/FileUpload.elm

- 660 行の自己完結コンポーネント（Model/Msg/update/view/subscriptions）
- D&D ゾーン + ファイル選択ダイアログ
- アップロード状態マシン: Pending → RequestingUrl → Uploading Float → Confirming → Completed / Failed
- `Http.track` によるアップロード進捗追跡（subscriptions 必要）
- バリデーション: `validateFile`（Content-Type, サイズ）、`validateFileCount`（ファイル数）
- テスト: `FileUploadTest.elm`（バリデーション 10 件）

### Phase 4: フロントエンド — 申請フォームへの FileUpload 統合

- `DynamicForm.elm`: `viewFields` に `(String -> Html msg)` コールバックパラメータ追加（Open-Closed Principle）
- `FormView.elm`: `viewFileField` で FileUpload コンポーネントをレンダリング
- `Types.elm`: `EditingState` に `fileUploads : Dict String FileUpload.Model` 追加、`initFileUploads` でファイルフィールドを自動検出
- `Update.elm`: `FileUploadMsg` ルーティング、`startFileUploads`（保存成功後にアップロード開始）、`validateFileFields` ヘルパー
- `New.elm`: `subscriptions` 追加（`Dict.map` + `Sub.batch` で全 FileUpload の進捗購読を集約）
- `Main.elm`: subscriptions に `WorkflowNewPage` の分岐追加
- `Validation.elm`: `validateFileField` 追加、`validateField` は File フィールドをスキップ

### Phase 5: フロントエンド — 詳細画面の添付ファイル表示とダウンロード

- `Detail/Types.elm`: `attachments : RemoteData ApiError (List Document)` フィールド、`GotAttachments`/`DownloadFile`/`GotDownloadUrl` メッセージ追加
- `Detail.elm`: ワークフロー取得後に添付ファイル一覧を非同期ロード、ダウンロードは Presigned URL を取得して `Ports.openUrl` で新タブオープン
- `Ports.elm`: `openUrl` ポート追加
- `main.js`: `window.open(url, "_blank")` ハンドラー追加

## 判断ログ

- DynamicForm の拡張方針: FileUpload への直接依存ではなく `(String -> Html msg)` コールバック注入を選択。Open-Closed Principle により、新しいフィールド型追加時に DynamicForm のコアを変更する必要がない
- アップロードタイミング: ファイル選択時の即時アップロードではなく、下書き保存後にアップロードを開始する設計を採用。既存 API が `workflow_instance_id` をアップロード時に要求するため
- ダウンロード方式: `Nav.load`（SPA リロード）ではなく Elm Ports + `window.open` を選択。ページ状態を保持するため
- FileError コンストラクタ引数の削除: elm-review の `NoUnused.CustomTypeConstructorArgs` 警告に従い、`InvalidType String` → `InvalidType` 等に簡略化。YAGNI に従い、エラー詳細表示が必要になった時点で再追加する
- `validateForm` のリファクタリング: `DynamicForm.extractFormFields` の重複呼び出しを排除し、`validateFileFields` ヘルパーに統合
