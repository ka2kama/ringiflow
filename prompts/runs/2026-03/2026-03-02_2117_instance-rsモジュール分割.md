# instance.rs モジュール分割

- 日時: 2026-03-02 21:17
- Issue: #1006
- PR: #1012

## 概要

ADR-062 に基づき、`instance.rs`（prod ~754行）を 4 ファイルに分割した。

## 判断ログ

### テストの配置先

テストコード（~638行）を各サブモジュールに分散させるか、親モジュールに保持するか検討した。

選択: 親モジュールに保持

理由:
- テストが `from_db()` と状態遷移メソッドの両方を使用している（ほぼ全ての状態遷移テストが `from_db()` で expected 値を構築）
- サブモジュールに分散させると、テスト内で `crate::workflow::*` のような長いパスが必要になる
- 完了基準は「prod コードが 500 行以下」であり、テストは対象外

## 成果

| ファイル | 行数 | 内容 |
|---------|------|------|
| `instance.rs`（親） | 245 prod + 637 test | 構造体定義, new(), アクセサ, re-exports, テスト |
| `instance/state.rs` | 128 | 型安全ステートマシンの状態型 |
| `instance/transitions.rs` | 262 | 8 つの状態遷移メソッド + can_edit |
| `instance/record.rs` | 188 | WorkflowInstanceRecord, from_db() |

## 適用パターン

ADR-043 の Rust モジュール分割パターン（`parent.rs` + `parent/` ディレクトリ）を適用。先行実例は `handler/workflow.rs`。

Rust 2018+ のモジュールシステムにより、`impl` ブロックを別ファイルに分散しても同一クレート内のため private フィールドにアクセス可能。
