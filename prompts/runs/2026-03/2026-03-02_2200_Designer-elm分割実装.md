# Designer.elm 分割実装

Issue: #1004
PR: #1013
日時: 2026-03-02

## 概要

Designer.elm（1965行）を ADR-062 の分割計画に基づいて 7 ファイルに分割した。
ADR-062 は 6 ファイル構成だったが、Elm の循環依存制約により Types.elm を追加した。

## 実装結果

| ファイル | 行数 | 責務 |
|---------|------|------|
| Designer.elm | 165 | 統合（init, update ルーティング, subscriptions, view レイアウト） |
| Types.elm | 117 | 共有型（Model, PageState, CanvasState, Msg） |
| Update.elm | 696 | メッセージ処理（updateLoaded, ヘルパー関数） |
| Canvas.elm | 659 | SVG キャンバス描画 |
| PropertyPanel.elm | 158 | プロパティパネル |
| Palette.elm | 90 | ステップパレット + アイコン |
| Toolbar.elm | 173 | ツールバー、メッセージ、バリデーション、ステータスバー |

合計: 2058行（元の 1965行から +93行。モジュール宣言・import 文の増加による）

## 判断ログ

### Types.elm の導入

ADR-062 計画にはなかったが、Elm のモジュール循環依存制約に対応するため Types.elm を追加した。
Update.elm が Msg と CanvasState を参照する必要があり、これらが Designer.elm にあると循環依存が発生する。
全サブモジュールが Types.elm から import する一方向の依存グラフにした。

### List.Extra への置き換え

Phase 2（Update.elm 抽出）で、ファイル内に定義されていた `removeAt`/`getAt` ヘルパーを
`List.Extra.removeAt`/`List.Extra.getAt` に置き換えた（重複排除）。

### ベースライン更新（FILE_SIZE_MAX_COUNT: 34 → 35）

元の Designer.elm（1965行）が500行超リストから消えた一方、Canvas.elm（659行）と Update.elm（696行）が
新たに追加されたため、500行超ファイル数が +1 となった。ベースラインを 34 → 35 に更新した。

## 技術的な知見

### Edit ツールの制約

大量のコード（数百行）を削除する際、Edit ツールの「一意な文字列マッチ」要件が障壁となった。
部分的な置換を繰り返すと孤立したコード断片が残る場合がある。
`sed -i '<start>,<end>d'` で行範囲を指定して一括削除する方が確実だった。

### Elm の import と elm-review

`StepType(..)` は型名 `StepType` とコンストラクタ（Start, Approval, End）の両方を import する。
型名を型注釈で使わず、コンストラクタも直接使わない場合、`StepType(..)` 自体が不要。
elm-review の `NoUnused.Variables` ルールがこれを検出する。
