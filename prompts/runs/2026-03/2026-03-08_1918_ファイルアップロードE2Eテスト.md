# ファイルアップロード E2E テスト追加（#1085）

## 概要

Issue #1085 に基づき、ファイルアップロード E2E テスト（E2E-007）を追加した。テスト追加の過程で、`confirmUpload` のデコーダーに既存バグを発見・修正した。

## 実施内容

### テストインフラ整備

- ファイル添付申請用のシードデータ（ワークフロー定義）を追加
- テスト用アップロードファイル（`tests/e2e/fixtures/test-upload.txt`）を作成
- `test-data.ts` に `FILE_UPLOAD_DEFINITION_ID` 定数を追加

### E2E テスト追加

2 つのテストを `file-upload.spec.ts` に作成:

1. ドキュメント管理画面: フォルダ作成 → ファイルアップロード → 一覧表示確認
2. ワークフロー申請: ファイル添付 → 申請 → 詳細画面で添付ファイル確認

### confirmUpload デコーダーバグ修正

`Api/Document.elm` の `confirmUpload` が `Document.decoder` を直接使用していたが、BFF レスポンスは `{"data": {...}}` 形式でラップされていた。`Decode.field "data" Document.decoder` に修正。

### E2E テスト突合表更新

E2E-007 のステータスを「カバー済み」に変更。サマリーのカウントを修正。

## 判断ログ

- ESM モジュール環境で `__dirname` が使えない問題 → `fileURLToPath(import.meta.url)` パターンを採用（既存の Playwright 設定が ESM のため）
- `getByRole("button", { name: "作成" })` が「フォルダ作成」にも部分マッチ → `exact: true` で完全一致に変更
- `confirmUpload` のデコーダーバグ発見: `page.route` インターセプトで API レスポンスが正常であることを確認し、DecodeError の原因を特定 → 調査記録に詳細を記録
