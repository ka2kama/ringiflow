# 2026-01-15_05_MCPサーバー導入見送り

## 概要

Claude Code における MCP（Model Context Protocol）サーバーの導入を検討し、現時点では導入しないことを決定。ADR として記録した。

## 背景と目的

ユーザーがこのリポジトリで推奨される MCP サーバーについて質問したことがきっかけ。MCP に詳しくないため、導入すべきかの判断材料を求めていた。

## 実施内容

1. **現状調査**: リポジトリ内の MCP 設定（`.mcp.json`）および関連ドキュメントを確認
2. **MCP の説明**: MCP とは何か、何ができるかをユーザーに解説
3. **導入判断**: 現在の開発環境（Bash + 標準ツール）で十分であり、導入不要と判断
4. **ADR 作成**: 決定内容を ADR-009 として文書化

## 成果物

### 作成ファイル

- `docs/04_ADR/009_MCPサーバー導入の見送り.md`

## 設計判断と実装解説

### MCP サーバーとは

MCP は Claude Code に追加機能を与える拡張機構。Anthropic 公式サーバーには以下がある:

| サーバー | 機能 |
|---------|------|
| `@anthropic/mcp-server-filesystem` | ファイルシステム操作 |
| `@anthropic/mcp-server-github` | GitHub API 連携 |
| `@anthropic/mcp-server-postgres` | PostgreSQL 操作 |

### 導入しない理由

1. **標準機能で代替可能**: Git は `git` コマンド、DB は `psql` コマンドで操作可能
2. **シンプルさの維持**: プロジェクト理念「KISS」に合致
3. **導入コストに見合わない**: 現時点で MCP でしか実現できないワークフローがない

### 再検討のトリガー

- 外部 SaaS（Slack, Notion 等）連携が必要になった場合
- GitHub API の頻繁な自動化が必要になった場合

## ユーザープロンプト（抜粋）

> このリポジトリで扱う、推奨かつ信頼度の高い MCP サーバがあれば教えてください。

> MCP に詳しくないので入れるべきかの判断がつきません。

## 学んだこと

- ツールは「必要になってから導入する」のが原則。複雑さを増やすだけの導入は避ける
- ADR は「採用した技術」だけでなく「採用しなかった判断」も記録する価値がある
