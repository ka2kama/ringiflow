# 2026-01-13: 学習支援方針策定セッション

## 概要

AI エージェントによる実装時に、コードだけでなく設計意図・代替案・トレードオフを含む解説を提供する方針を CLAUDE.md に明文化した。

## 背景

本プロジェクトは以下の2つの目的を持つ:

1. AI エージェントによる実装の実験場
2. プロジェクトオーナーの技術的学習の場

AI が生成したコードをオーナーが理解し、知識として定着させる必要があるため、実装の背景や設計判断の根拠を含む解説が求められた。

## 要件確認

方針を明文化する前に、以下の3点を確認した:

| 項目 | 選択肢 | 決定 |
|------|--------|------|
| 解説の詳細度 | 常に詳細 / 複雑な箇所のみ / 質問時のみ | 複雑な箇所のみ |
| 技術スタックの経験レベル | 初学者 / 中級者 / スタックにより異なる | 中級者 |
| 注力領域 | アーキテクチャ / Rust / Elm・FP / 全般 | アーキテクチャ設計、TEA パターン、全般 |

## 実施内容

### CLAUDE.md への「学習支援方針」セクション追加

追加位置: 「基本方針: 暗黙知ゼロ」セクションの直後

追加内容:

| サブセクション | 内容 |
|---------------|------|
| 解説のタイミング | 設計判断を伴う箇所（アーキテクチャ決定、パターン選択、非自明なロジック、トレードオフ）に限定 |
| 解説の内容 | 意図 → 代替案 → トレードオフ → 関連知識 の4点構成 |
| 想定レベル | 中級者（基本構文は省略、応用・ベストプラクティス観点で解説） |
| 注力領域 | アーキテクチャ設計、Elm/TEA パターン、ソフトウェア設計原則 |

## 使用したプロンプトパターン

### 要件確認を促す依頼

```
[目的・背景を説明した上で]
この旨を CLAUDE.md に指示として残してください。
必要な情報があれば確認してください。
```

ポイント:
- 背景（なぜ必要か）を明示する
- 「必要な情報があれば確認」で双方向のやりとりを促進する
- AI が一方的に仮定せず、適切な粒度で要件を確認できる

## 再利用可能なパターン

### AI への期待値を明文化するフロー

1. 背景・目的を伝達（なぜその対応が必要か）
2. 「必要な情報があれば確認」で要件確認を促す
3. AI からの質問に回答（選択肢形式が効率的）
4. 確認結果を踏まえて明文化

### 解説の標準構成

「意図 → 代替案 → トレードオフ → 関連知識」の4点構成は、技術的な説明の標準フォーマットとして汎用的に活用可能。
