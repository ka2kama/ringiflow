## 背景と目的

## 設計判断

### User Project vs Repository Project

GitHub Projects v2 では User/Organization レベルの Project が推奨されている。
Repository Project（Classic）は旧式で機能が限定的。

User Project をリポジトリに紐づけることで、リポジトリの Projects タブからもアクセス可能。

### rules ファイルへの分離について

CLAUDE.md の肥大化を検討したが、Issue 駆動開発と Git 操作ルールは常に適用される必要があるため、CLAUDE.md に残すことにした。

rules ファイルは `paths` で指定されたファイル編集時にのみ適用されるため、常に適用すべきルールには不向き。

## 学んだこと

- GitHub Projects v2 は User/Organization レベルで作成し、リポジトリに紐づけるのが推奨
- `gh project` コマンドで CLI から Project を操作可能
- Milestone は Issue のグルーピングに使い、Phase ごとの進捗管理に適している
- 設計書は「どう作るか」を定義する場所であり、進捗管理は別の仕組みで行うべき
