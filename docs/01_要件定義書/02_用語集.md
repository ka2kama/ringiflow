## [GLOSS-01] 用語集

| 用語 | 説明 |
|------|------|
| テナント | 本システムを利用する組織（原則: 会社）単位。監査・課金・データ分離の境界。 |
| ワークフロー | 業務プロセスを定義したもの |
| ワークフローインスタンス | 実行中のワークフロー |
| ACU | Aurora Capacity Unit（Aurora Serverless のリソース単位） |
| AZ | Availability Zone（AWS のデータセンター群） |
| Blue/Green | 新旧環境を並行稼働させるデプロイ手法 |
| RBAC | Role-Based Access Control（役割ベースアクセス制御） |
| ABAC | Attribute-Based Access Control（属性ベースアクセス制御） |
| SLA | Service Level Agreement（サービスレベル合意） |
| RTO | Recovery Time Objective（目標復旧時間） |
| RPO | Recovery Point Objective（目標復旧時点） |
| OTEL | OpenTelemetry（可観測性フレームワーク） |
| イベントソーシング | 状態変更をイベントの履歴として保存するパターン |
| CQRS | Command Query Responsibility Segregation（コマンドとクエリの責務分離） |
| Aggregate | ドメイン駆動設計における一貫性境界の単位 |
| Projection | イベントから Read Model を構築する処理 |
| Read Model | 読み取り専用に最適化されたデータモデル |
| Event Store | イベントを永続化するデータストア |
| Saga | マイクロサービス間の分散トランザクションパターン |
| 補償トランザクション | Saga において失敗時に実行される取り消し処理 |
| Outbox パターン | DB書き込みとイベント発行の原子性を保証するパターン |
| CDC | Change Data Capture（データ変更の検出・伝播） |
| 結果整合性 | 一定時間後に整合性が保証されるモデル |
| 冪等性 | 同じ操作を複数回実行しても結果が変わらない性質 |
| WebSocket | 双方向リアルタイム通信プロトコル |
| Pub/Sub | Publish/Subscribe（発行/購読型メッセージング） |
| パーティション | Kafka におけるトピックの分割単位 |
| コンシューマーグループ | Kafka メッセージを分散処理するコンシューマーの集合 |
| BFF | Backend for Frontend の略。フロントエンド専用のバックエンドレイヤー |
| Ports | Elm アプリケーションと外部 JavaScript の通信インターフェース |
| PITR | Point-in-Time Recovery。任意時点へのデータベース復元機能 |
| DLQ | Dead Letter Queue。処理失敗メッセージの退避先キュー |
| WCAG | Web Content Accessibility Guidelines。ウェブアクセシビリティのガイドライン |
| i18n | Internationalization（国際化）の略称。多言語対応の仕組み |
| ICU MessageFormat | 国際化対応のメッセージフォーマット仕様 |
| RFC 7807 | HTTP API のエラーレスポンス形式を定義した標準（Problem Details） |
| Runbook | 運用手順書。障害対応や定型作業の手順を記載したドキュメント |
| SBOM | Software Bill of Materials。ソフトウェア部品表 |
| DAST | Dynamic Application Security Testing。動的アプリケーションセキュリティテスト |
| SAST | Static Application Security Testing。静的アプリケーションセキュリティテスト |
| PGO | Profile-Guided Optimization。プロファイルに基づく最適化 |
| ファセット | 検索結果をカテゴリ別に分類・絞り込む機能 |
| スニペット | 検索結果で表示する本文の抜粋 |
| IdP | Identity Provider。認証を担当する外部サービス（例: Okta, Azure AD） |
| OIDC | OpenID Connect。OAuth 2.0 をベースにした認証プロトコル |
| SAML | Security Assertion Markup Language。XML ベースの認証連携規格 |
| PKCE | Proof Key for Code Exchange。認可コード横取り攻撃を防ぐ仕組み |
| TOTP | Time-based One-Time Password。時刻ベースのワンタイムパスワード |
| MFA | Multi-Factor Authentication。多要素認証 |
| HTTPOnly Cookie | JavaScript からアクセスできない Cookie。XSS 対策に使用 |
| SameSite Cookie | クロスサイトリクエストで送信を制限する Cookie 属性 |
| CSRF | Cross-Site Request Forgery。ユーザーの意図しないリクエストを送信させる攻撃 |
| XSS | Cross-Site Scripting。悪意のあるスクリプトを埋め込む攻撃 |
| CORS | Cross-Origin Resource Sharing。異なるオリジン間でのリソース共有を制御 |
| TLS | Transport Layer Security。通信を暗号化するプロトコル |
| AES-256 | Advanced Encryption Standard 256bit。対称暗号方式 |
| KMS | AWS Key Management Service。暗号化キーを管理するサービス |
| WAF | Web Application Firewall。Web アプリへの攻撃を検知・防御 |
| SLO | Service Level Objective。サービス品質の目標値 |
| SLI | Service Level Indicator。サービス品質を測定する指標 |
| RESTful | リソース指向の API 設計スタイル |
| JSON Schema | JSON データの構造を定義・検証する標準仕様 |
| OpenAPI | REST API の仕様を記述する標準フォーマット（旧 Swagger） |
| サーキットブレーカ | 障害時にリクエストを遮断し、連鎖障害を防ぐパターン |
| バックプレッシャ | 下流が処理しきれない場合に上流の流量を抑える仕組み |
| レート制限 | 単位時間あたりのリクエスト数を制限する仕組み |
| スロットリング | リクエストを制限し過負荷を防ぐこと |
| VPC | Virtual Private Cloud。AWS 上の仮想ネットワーク |
| NAT Gateway | Private サブネットからの外向き通信を可能にするゲートウェイ |
| VPC エンドポイント | VPC 内で AWS サービスに接続するためのエンドポイント |
| ECR | Elastic Container Registry。Docker イメージを保存するレジストリ |
| Secrets Manager | 機密情報を安全に保管・取得する AWS サービス |
| SSM | AWS Systems Manager。パラメータストア等を提供 |
| ACM | AWS Certificate Manager。SSL/TLS 証明書を管理 |
| Glacier | S3 の低コストストレージクラス。長期保存用 |
| TTL | Time To Live。データの有効期限 |
| RLS | Row Level Security。行単位でアクセス制御する DB 機能 |
| ETL | Extract-Transform-Load。データの抽出・変換・取り込み処理 |
| DWH | Data Warehouse。分析用に整理されたデータベース |
| APM | Application Performance Monitoring。アプリの性能監視 |
| PagerDuty | インシデント管理・オンコール通知サービス |
| UAT | User Acceptance Test。ユーザー受け入れテスト |
| GitHub Flow | main からフィーチャーブランチを作成し PR でマージするワークフロー |
| Expand/Contract | DB マイグレーションの後方互換を保つパターン（追加→移行→削除） |
| Zero Trust | 「何も信頼しない」を前提とするセキュリティモデル |
| マイクロセグメンテーション | ネットワークを細かく分割しサービス間通信を個別制御する方式 |
| 楽観ロック | 更新時にバージョン番号をチェックし競合を検出する方式 |
| 指数バックオフ | リトライ間隔を指数的に増やす方式（1秒→2秒→4秒...） |
| ポストモーテム | 障害発生後の振り返りと再発防止策の策定 |
