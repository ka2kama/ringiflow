# ナレッジベース

## 概要

ナレッジベースは、プロジェクトで使用する技術・ツール・概念についての知識を整理したドキュメントです。
「暗黙知ゼロ」の理念に基づき、チームメンバー（人間・AI 問わず）が技術を理解するための参照資料として機能します。

技術的な知識（旧・技術ノート）と学習の記録（旧・学習ノート）を統合し、カテゴリ別に管理しています。

## ADR との違い

| 観点 | ADR | ナレッジベース |
|------|-----|---------------|
| 焦点 | なぜその決定をしたか（WHY） | その技術は何か、どう使うか（WHAT/HOW） |
| 内容 | 意思決定の記録、代替案との比較 | 機能説明、使い方、概念整理、学習の記録 |
| 更新 | 原則不変（決定の記録） | 知識が増えたら更新可 |
| 例 | 「なぜ SQLx を選んだか」 | 「sqlx-cli の使い方」 |

## ナレッジの書き方

### ファイル命名規則

```
docs/06_ナレッジベース/<カテゴリ>/技術名.md
```

例:
- `rust/sqlx-cli.md`
- `elm/Elmアーキテクチャ.md`
- `architecture/BFFパターン.md`

新しいカテゴリが必要な場合はサブディレクトリを追加し、このREADME のカテゴリ一覧も更新する。

### 推奨セクション

1. 概要: その技術が何か、何のために使うか
2. 主な機能: 機能一覧や API の概要
3. 使い方: 基本的な使用例
4. プロジェクトでの使用箇所: どこでどう使っているか
5. 関連リソース: 公式ドキュメント、参考記事へのリンク

### 書くべきこと

- ツールのインストール方法・基本コマンド
- 概念の説明（TEA パターン、CQRS など）
- ベストプラクティス、Tips
- トラブルシューティング
- 学習で得た気づき・理解

### 書かないこと

- 技術選定の理由（→ ADR へ）
- プロジェクト固有の設計詳細（→ 設計書へ）
- 操作手順（→ 手順書へ）
