# 実装解説

AI エージェント主導で実装した内容の詳細解説を記録する。
オーナーのレビュー・学習を支援することが目的。

## 目的

1. レビュー支援: 実装内容を理解し、品質を検証できる
2. 学習効果: 設計判断の理由、代替案、トレードオフを学べる
3. 知識の蓄積: 将来の参照用として設計意図を残す

## ディレクトリ構造

```
docs/07_実装解説/
├── README.md                           # 本ファイル
├── 01_認証機能/                        # Issue/機能単位
│   ├── 00_概要.md                      # 機能全体の概要（任意）
│   ├── 01_Phase1_UserRepository.md     # Phase 単位の解説
│   ├── 02_Phase2_PasswordHasher.md
│   └── ...
└── 02_承認フロー/
    └── ...
```

## ファイル命名規則

```
NN_PhaseN_<コンポーネント名>.md
```

- `NN`: サブディレクトリ内の連番（`01_`, `02_`, ...）
- `PhaseN`: 実装フェーズ番号
- `<コンポーネント名>`: 主要な実装対象

例: `01_Phase1_UserRepository.md`

## 解説ドキュメントの構成

ドキュメントは「何を実装したか」を前半に、「設計解説」を後半に配置する。

### 前半: 実装内容（レビュー用）

| セクション | 内容 |
|-----------|------|
| 概要 | この Phase で何を実装したか |
| └ 対応 Issue | GitHub Issue へのリンク |
| 設計書との対応 | 設計書のどのセクションを実装したか（リンク付き） |
| 実装したコンポーネント | ファイル一覧と責務（リンク付き） |
| 実装内容 | 主要な型・メソッドの説明 |
| テスト | テストケース一覧と実行方法 |
| 関連ドキュメント | 設計書、技術ノートへのリンク |

### 後半: 設計解説（学習用）

「設計解説」という見出しの下に、番号付きでトピックを列挙する。

各トピックの構成:
- **場所**: ソースコードへのリンク（行番号付き）
- **コード例**: 該当コードの抜粋
- **なぜこの設計か**: 設計判断の理由
- **代替案**: 検討した他の選択肢とトレードオフ

## 他のドキュメントとの関係

| ドキュメント | 役割 | 実装解説との違い |
|-------------|------|-----------------|
| 設計書 | 事前の設計仕様 | 実装解説は「実際にどう作ったか」 |
| ADR | 意思決定の記録 | 実装解説は「どう実装したか」も含む |
| 技術ノート | 汎用的な技術知識 | 実装解説はプロジェクト固有の実装 |
| セッションログ | 作業記録 | 実装解説は体系的な解説 |

## 作成タイミング

- 各 Phase の実装が完了した時点で作成する
- セッション終了時にまとめて作成しても良い
- 遡って作成することも可能
