# document vs documentation — 可算と不可算で変わる視点

日本語ではどちらも「ドキュメント」と訳せるが、英語では視点が異なる。

## 比較

| | document(s) | documentation |
|---|---|---|
| 可算性 | 可算名詞 | 不可算名詞 |
| 視点 | 個々のファイル・書類 | 体系・プロセスとしてのまとまり |
| 例 | a design document, three documents | API documentation, good documentation |

## 使い分け

### documents — 具体的な個体を意識

```
"I updated the design documents."
→ 設計書ファイルを（複数）更新した

"There are 10 documents in this folder."
→ このフォルダに 10 個のファイルがある
```

個別のファイルを数えられる・指し示せるときに使う。

### documentation — 体系・全体像を意識

```
"This library lacks documentation."
→ このライブラリはドキュメントが不足している（全般的に）

"Read the API documentation."
→ API ドキュメント（体系として）を読んで

"Add documentation for this feature."
→ この機能のドキュメントを追加して（文書化して）
```

特定のファイルではなく、説明・記述の全体的な充実度を語るときに使う。

## 開発でよく使う表現

| 英語 | 意味 | 使っている語 |
|------|------|-------------|
| Add docs if needed | 必要ならドキュメントを追加して | docs（document の略・複数形） |
| Add documentation if needed | 必要ならドキュメントを整備して | documentation |
| Update the design document | 設計書を更新して | document（単数） |
| Improve the documentation | ドキュメントを改善して | documentation |

`docs` は `documents` のカジュアルな略語で、ソフトウェア開発では非常に一般的。

## -ation 接尾辞のパターン

`-ation` が付くと「個別の物」から「体系・プロセス」に抽象化される傾向がある:

| 具体（可算） | 抽象（不可算） |
|-------------|---------------|
| a document | documentation |
| an instrument | instrumentation |
| — | equipment（equip + ment） |
| an implement | implementation（※ 可算の場合もある） |

## 学び

日本語の「ドキュメント」は両方の意味をカバーするため、英語で書く際に混同しやすい。「個々のファイルか、体系としてか」を意識すると使い分けられる。
